# 原子实施计划与进度表

## 1. 使用说明

- 本计划是唯一开发顺序依据；架构范围见`docs/architecture.md`，强制流程见`docs/development-workflow.md`。
- `D`后缀表示详细设计项，`I`后缀表示实现项；任何`I`项必须等待对应`D`项完成并推送。
- 每次只允许一个计划项处于“进行中”。
- 每个原子项完成验证后先提交推送，再回写状态、提交号和验证证据并再次推送。
- 状态只能使用：待开始、进行中、已验证、已提交、已完成、阻塞。

## 5. 计划变更规则

- 计划顺序、范围或验收条件变化必须先修改本文件并提交；
- 新需求只能追加为新的设计项和实现项，不得直接插入当前实现；
- 因环境缺失阻塞时记录缺失组件、检查结果和恢复条件；
- 已完成项不得改写提交号，后续修订使用新的原子项。
